Was uns bewegt

Allow me in the following some notes, using Reginald.

Note 1

The MagicNav element always requires the header element. It can be set to be fixed or as floating item.

Note 2

The additional elements can be found in the LIST or library panel. They're to be seen as placeholder code ready to be filled with life.

Note 3

Watch out for the properties-panel. In there you'll find the settings. Enough stuff for the very basics - the rest is designer's land.

Sitemap

The Top-Nav is dediacted to drive through the sections of the current page. But assumed, you have a growing project, with more pages to add, the Top Nav would be too crowded. Drop Downs and the like sucks. So the Sitemap is to explore the Site - and will link to the next following pages.
